# Future-Spark-Institute-AI-Creation-Digital-Gallery（未来火种研究院 · AI造物数字展厅）
# **一、项目名称、目标与背景**
                 
## **项目名称：**

未来火种研究院 · AI造物数字展厅

## **项目目标：**

搭建一个运行于云端的三维数字展厅，让参观者无论身处何地，都能以沉浸式、交互式的方式了解青少年自主设计的科技硬件作品，感受每一件作品背后的探索过程与创造精神。

## **项目背景：**

"未来火种研究院"是一个面向青少年的AI原生科技创作教育项目。这个 AI 造物数字展厅，由孩子们深度参与共建——他们既是作品的创作者，也是展厅的建设者。我们希望这里不只是展示作品的地方，而是孩子们相遇的作品空间，让一个孩子的创造启发另一个孩子，让更多青少年在这里看见彼此、互相点燃。

## 要解决的问题：

传统的作品展示方式主要存在以下局限：

| 展示方式         | 存在问题                  |
| ------------ | --------------------- |
| 文字 / 图片 / 视频 | 信息维度单一，缺乏立体感与交互性      |
| 线下实物展厅       | 搭建成本高，需持续人力运营，地域限制大   |
| 现场讲解         | 一对多模式下，参观者的个性化问题容易被忽略 |

以上方式均难以完整呈现一件作品从构思到落地的完整脉络，参观者的参与感和信息获取效率也相对有限。

# **二、作品描述**

## **作品形式：**

本项目以 NVIDIA Spark 为算力底座，使用 Isaac Sim 搭建三维虚拟展厅，展厅内陈列了多款青少年自制硬件作品：如专注喵、静习宝、智能碗、小和机器人。展厅内设有：

- 每件产品的三维实物模型

- 产品专属介绍视频

- 根据产品创作者形象生成的3D虚拟讲解员


## **核心亮点：**

1. **与创作者面对面交互**

  每件作品旁边站着的讲解员，就是这件作品的设计师孩子本人——用他/她真实形象生成的 3D 数字人。参观者可以直接问他"你为什么要做这个"、"这个按钮是干什么的"，每一次提问，都是一次参观者与创作者真实交流。

2. **看见一件作品的完整故事**

  三维模型、介绍视频、文字说明、实时问答四种形式并存，参观者可以从多个角度了解这个孩子为什么做这个、怎么做出来的、遇到过什么问题——不只是"这是什么产品"，而是"这个孩子是怎么想的"。

3. **云端可访问，零地域限制**

  展厅运行在 DGX Spark 上，后期支持远程进入，参观者无需到场，让更多人有机会看见这些孩子的创造。

## **作品功能概览：**

- 三维虚拟展厅浏览（基于 Isaac Sim）

- 与孩子设计师数字人一对一对话（基于 RAG 知识库，回答来自创作者本人的设计记录）

- 产品介绍视频播放（AI 文生视频）

- 3D 产品模型展示

# **三、技术实现方案、架构设计与NVIDIA工具应用**

## **3.1 整体架构概览**

本项目采用 "本地大模型 + RAG 知识增强 + 三维虚拟展厅" 的双系统融合架构，全部运行于 NVIDIA DGX Spark（GB10）本地，不依赖任何云端 API，数据全程本地处理，保障展厅运行的离线性与数据隐私安全。

![项目演示图](https://github.com/xiaoxiaokangkang/Future-Spark-Institute-AI-Creation-Digital-Gallery/raw/images/ArchitectureOverview.png)

## **3.2 AI 智能问答系统（RAG 模块）**

### **知识库构建：**

- 收集 22 个青少年作品的完整资料文档

- 将资料切分为 1214 个文本块，并通过 qwen3-embed:0.6b-q8 模型生成对应的向量 Embedding

- 构建双层检索机制：关键词匹配（精确定位）+ 向量索引（语义理解）相结合，知识库检索准确率达 97.6%


### **模型部署：**

- 使用 Ollama 推理框架（CUDA 13.0） 在 DGX Spark 上部署 Qwen3-8B 对话大模型

- 用户提问 → 检索知识卡片构建上下文 → 大模型基于上下文生成回答

- 端到端响应时间：7～8 秒

| 优化方向    | 具体措施                           | 效果                               |
| ------- | ------------------------------ | -------------------------------- |
| 内存占用优化  | 采用 Q4_K_M 量化，将模型压缩至 ~5GB       | 充分利用 GPU 统一内存带宽（LPDDR5X 273GB/s） |
| 输出稳定性优化 | 设计多级输出清洗函数（答案提取 → 指令过滤 → 去重截断） | 有效解决 8B 小模型输出自我指令、内容重复等问题        |
| 指令遵循优化  | 简化 Prompt 格式，降低小参数模型的理解负担      | 提升回答的针对性与可读性                     |

  实测推理速度达 ~44 tok/s，接近 LPDDR5X 273GB/s 带宽理论极限。

## **3.3 三维数字展厅系统（Isaac Sim 模块）**

### **场景制作流程：**

1. 在 Unreal Engine 5.2 中完成展厅场景的三维建模与设计

2. 将场景导出为 OpenUSD 格式（NVIDIA 通用场景描述标准）

3. 导入 Isaac Sim 5.1 进行本地渲染与运行


### **平台运行：**

- Isaac Sim 5.1 在 DGX Spark（aarch64 架构）上原生支持运行

- 渲染采用 RTX Renderer 实时光线追踪，呈现高质量展厅照明与材质效果

- 场景中集成 PhysX 物理仿真，支持展厅内物体的真实物理交互

## **3.5 技术创新点**

1. **消费级边缘设备上的完整 RAG 系统落地**

在 DGX Spark GB10 这一边缘计算设备上，实现了从知识库构建、向量检索到大模型推理的完整 RAG 问答链路，推理速度达 ~44 tok/s，验证了边缘设备承载完整 AI 问答系统的可行性。

2. **Isaac Sim 的跨场景应用探索**

Isaac Sim 原本定位于机器人仿真训练场景，本项目将其引入教育展示与数字人展厅场景，在 DGX Spark aarch64 平台上完成部署运行。这可能是国内首个在 DGX Spark aarch64 平台上运行 Isaac Sim 5.1 的非机器人应用案例。

3. **端到端本地算力**

展厅渲染、大模型推理、RAG 检索、TTS 语音合成全部在 DGX Spark 本地完成，实现无网络依赖、低延迟、数据不出本地的完整体验。

4. **孩子的知识库，由孩子喂养**

RAG 知识库的内容完全来自孩子自己的创作记录，让数字人能回答"这个按钮是我想了三个版本才定下来的"这类真实问题，而不是官方介绍。知识库可持续扩展，随着作品增加自动成长。

# **五、未来展望**

## **5.1 开放公开访问**

这次我们以视频形式提交演示，但核心技术路线已经完整跑通：全部算力在 DGX Spark 本地运行，Isaac Sim 实时渲染，RAG 实时问答。通过 WebUSD 或 Omniverse Cloud 让任何人点链接就能进入展厅，无需下载任何软件。

## **5.2 分阶段迭代计划**

###  **阶段一：**XR设备适配**

OpenUSD 格式天然支持空间计算设备，计划适配 Apple Vision Pro等设备，让参观者真正"走进"展厅，为参访者提供空间沉浸式浏览体验，进一步拉近与作品之间的距离。

###  **阶段二：数字人讲解升级**

展品讲解功能将进一步优化——由产品创作者本人的数字人形象出镜讲解，替代现有方式。参访者可以直接"面对"创作者了解作品的设计初衷、制作过程与使用方式，使讲解更具真实感与个人色彩。

## **5.3 教育价值延伸**

这个 AI 造物数字展厅不只是为了展示产品，更是为了让前来参观的孩子看见：同龄人如何从生活中的一个小问题出发，一步步动手做出完整的解决方案。

通过参观活动，我们希望孩子能被展示中的产品与故事点燃，产生我也能跟着他们一起做，我也要要去试一试的想法。我们想传递的理念是，在 AI 时代，每个有想法的孩子都可以走上这条创造之路。

展厅会持续收录新作品，成为一个不断生长的青少年科创档案馆。让更多有创造力的青少年拥有展示自己作品的数字展位。
